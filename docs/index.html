Plain-Language Guide

如果你不是工程師，RDE 到底會幫你做什麼？

你可以把 RDE 想成一位會留下完整工作紀錄的資料分析助理。它不只給答案，也會保存每一步為什麼這樣做、用了哪些資料、是否改過方法、最後報告能不能被審查。

先確認資料能不能用

RDE 會先檢查檔案格式、大小、是否有疑似個資、有哪些工作表或欄位。這一步像是收件與初步驗收，避免一開始就把不該分析的資料丟進模型。

把欄位翻成研究語言

資料裡可能有年齡、分組、治療方式、結果、時間、操作者、檢驗值。RDE 會建立 schema，並把欄位對應到「暴露、結果、共變數、分組、時間軸」等研究角色。

先做計畫，再跑統計

RDE 會先產生分析計畫，讓使用者確認主要問題、主要結果、要比較的組別、缺失值策略與圖表。計畫鎖定後才進入正式分析。

自動選擇合適方法

例如連續變數可能用 t-test 或 Mann-Whitney，類別變數可能用卡方或 Fisher；RDE 會根據欄位型態、組數、常態性與樣本狀況選擇方法。

報告不是只有 p 值

完整報告會包含資料概況、Table 1、比較結果、圖表、模型、敏感度或補充分析、方法限制、decision log 與 deviation log。

不把結果包裝成過度承諾

如果資料不足、方法不適合、缺少必要 artifact，RDE 會標記 report readiness 缺口，而不是把未完成結果硬說成 production-ready。

使用者通常會看到的分析旅程

同一份資料可以只做快速概況，也可以走完整審計流程。

1. 上傳或指定資料

RDE 先掃描資料來源，確認格式、大小與疑似個資。如果需要處理敏感資料，必須明確知道風險並用指定參數覆蓋。

2. 建立資料字典

系統整理欄位型態、缺失率、唯一值、描述統計、可能的群組欄位與 outcome 欄位，讓後續方法不是憑猜測。

3. 確認研究問題

使用者確認「我要比較什麼」、「主要結果是什麼」、「哪些變數可能是干擾因子」。這一步避免 agent 把錯的欄位當成主要結論。

4. 鎖定分析計畫

RDE 產生候選分析、方法學審查與執行順序。使用者確認後，Phase 8 才能照計畫執行。

5. 產生可審查報告

結果會被彙整成報告，並附上圖表、來源 artifact、決策紀錄與 audit。若後來改方法，偏離原因會寫入 deviation log。

Statistics In RDE

RDE 目前能做的基礎與進階統計

下面用「你想回答的問題」來說明，而不是用工程模組分類。RDE 會依資料型態、樣本數、分組、常態性與計畫內容選擇合適方法。

這份資料長什麼樣？

描述統計、缺失值、離群值、分布形狀、欄位型態、資料品質評估，以及 Table 1。Table 1 可用來整理不同組別的基線特徵。

profile_datasetassess_qualitygenerate_table_one

兩組或多組是否不同？

連續變數可依情境使用 t-test、Mann-Whitney、ANOVA、Kruskal-Wallis；類別變數可用卡方或 Fisher。重複測量或配對資料可用 paired t-test、Wilcoxon 或 Friedman。

compare_groupsrun_repeated_measures

變數之間有沒有關係？

RDE 可做 Pearson、Spearman、point-biserial correlation，也會在相關矩陣中提醒共線性風險，避免把高度重複的變數塞進模型。

correlation_matrixVIF

調整其他因素後還成立嗎？

如果問題不是單純比較，而是想控制年齡、共病、嚴重度等干擾因子，RDE 可用 logistic regression、multiple regression 或 GLM 做調整模型。

run_advanced_analysislogistic_regressionmultiple_regression

模型判斷能力如何？

二元結果可以評估 ROC/AUC；非顯著結果或設計階段可用 power analysis 輔助判斷樣本數或檢定力是否不足。

roc_aucpower_analysis_advanced

時間到事件資料怎麼看？

若資料有追蹤時間與事件欄位，RDE 可產生 Kaplan-Meier 摘要；在資料條件可行時，可跑 Cox regression 做調整後的存活分析。

kaplan_meiercox_regression

兩組本來就不公平怎麼辦？

對觀察性資料，RDE 可用輕量 propensity score 估計分組機率、檢查 overlap、產生權重或簡單 matching 摘要，幫助討論組間平衡。

propensity_scorebalance diagnostics

學習曲線或操作熟練度怎麼看？

如果資料有操作者、嘗試順序與成功/失敗，RDE 可用 learning-curve CUSUM 探索表現是否隨經驗改善。

learning_curve_cusum
重要限制： RDE 不會把所有統計方法一次全跑。它會先讓使用者確認研究問題與分析計畫，再選擇合適方法。這樣可以降低多重比較、假陽性與方法漂移。

圖表會搭配問題產生

RDE 目前支援常見探索圖表：histogram、boxplot、violin、scatter、bar、heatmap、line、paired plot。完整報告會盡量讓圖表跟表格、模型與文字解釋互相對應。

Auto Research

Auto Research 是受控探索，不是自動改結論

RDE 的 Auto Research 概念是：在主要分析計畫鎖定後，讓 agent 在明確邊界內探索額外假設，留下分支紀錄、實驗結果、評估與是否值得納入計畫的理由。

它像研究助理的白板

主要計畫仍是主線。Auto Research 會另外開探索分支，例如「是否需要調整模型」、「是否值得做 propensity score」、「某個子群是否需要敏感度分析」。

它只能在 gate 後執行

分支探索需要 locked plan 與 readiness 通過。也就是說，不能在還沒理解資料、還沒確認研究問題時就自動亂跑。

它有預算與停止機制

Auto Research queue 會記錄工作項目、進度、失敗預算、停止與恢復事件，避免探索無限制擴張。

它不會直接改主結論

探索分支若想升級成正式 plan amendment，需要 evidence、evaluate_branch、promotion gate 與使用者確認。

它會留下完整紀錄

每個 branch、experiment、evaluation、promotion review 都會寫入 project artifacts，報告可以引用，也可以在 audit 中檢查。

它適合找補充分析

常見用途包括：補一個調整模型、做 ROC/AUC、檢查 propensity balance、探索學習曲線、找需要敏感度分析的變數。

Auto Research 的安全流程

這是目前 branch tools 與 agent-control manifest 所描述的控制邏輯。

Open branch

先記錄 hypothesis、reason、相關變數與風險等級。這一步只是開探索分支，不改 locked plan。

Suggest experiments

根據 schema、locked plan 與變數角色提出可能的 branch experiments，例如 adjusted model、ROC/AUC 或 propensity score。

Run queue

Auto Research run 會建立 work queue、work events、budget state 與 progress events，可停止、恢復或逐項執行。

Evaluate branch

分支需要有 live evidence artifact、樣本數、效果或模型指標、 uncertainty 或 diagnostic metric，才有評估基礎。

Promote only with confirmation

要把分支升級成 plan amendment，必須通過 audit gate、不是 stale evaluation、且需要使用者確認。它不能自動合併成主要結論。

一句話理解： Auto Research 不是「讓 AI 自己研究完並宣布答案」，而是「讓 AI 在已鎖定的研究計畫旁邊開受控分支，收集補充證據，最後由 gate 與使用者決定能不能納入」。

Plain-Language Guide

What does RDE do for a non-engineer?

Think of RDE as a data-analysis assistant that keeps receipts. It does not only produce an answer; it preserves why each step happened, which data supported it, whether the method changed, and whether the final report is ready to trust.

It checks whether the data can be used

RDE reviews file format, file size, suspected PII, workbooks, sheets, columns, and basic quality before analysis begins.

It turns columns into research concepts

Columns such as age, group, treatment, outcome, time, operator, and lab values are mapped into roles such as exposure, outcome, covariate, group, or time axis.

It plans before running statistics

The user confirms the research question, endpoints, groups, missing-data strategy, analysis families, and figures before the plan is locked.

It chooses methods based on data shape

Continuous variables may use t-test or Mann-Whitney; categorical variables may use chi-square or Fisher. The choice depends on type, group count, normality, and sample size.

It reports more than p-values

A full report includes data overview, Table 1, comparisons, figures, models, sensitivity or branch analyses, limitations, decision logs, and deviation logs.

It avoids overclaiming

If evidence is missing, a method is unsuitable, or required artifacts are absent, RDE reports readiness gaps instead of calling the output production-ready.

Statistics In RDE

What statistics can RDE currently run?

This page explains capabilities by user question. RDE chooses methods from the locked plan and the data structure, not by blindly running every possible test.

What does the dataset look like?

Descriptive statistics, missingness, outliers, variable types, quality assessment, and Table 1 for baseline characteristics.

profile_datasetassess_qualitygenerate_table_one

Are two or more groups different?

Continuous variables can use t-test, Mann-Whitney, ANOVA, or Kruskal-Wallis. Categorical variables can use chi-square or Fisher. Paired/repeated data can use paired t-test, Wilcoxon, or Friedman.

compare_groupsrun_repeated_measures

Are variables related?

RDE can run Pearson, Spearman, point-biserial correlation, and correlation matrices with collinearity warnings.

correlation_matrixVIF

Does the finding hold after adjustment?

For confounding control, RDE supports logistic regression, multiple regression, and GLM when the data structure is suitable.

run_advanced_analysislogistic_regressionmultiple_regression

How well does a model discriminate?

Binary outcomes can be evaluated with ROC/AUC, and power analysis can help interpret non-significant findings or sample-size concerns.

roc_aucpower_analysis_advanced

What about time-to-event data?

When time and event columns exist, RDE can generate Kaplan-Meier summaries and Cox regression when feasible.

kaplan_meiercox_regression

What if groups are imbalanced?

For observational data, lightweight propensity scoring can estimate treatment probability, inspect overlap, and summarize weights or simple matching.

propensity_scorebalance diagnostics

Can it examine learning curves?

If data includes an operator, trial order, and success/failure, RDE can explore learning-curve CUSUM patterns.

learning_curve_cusum
Important limit: RDE does not run every method at once. It confirms the research question and analysis plan first to reduce false positives, multiple comparisons, and method drift.

Auto Research

Auto Research is controlled exploration, not automatic conclusion rewriting

RDE's Auto Research concept lets an agent explore extra hypotheses beside the locked primary plan, while preserving branch records, experiment results, evaluations, and promotion decisions.

A research assistant's side board

The main plan remains the main line. Auto Research opens exploratory branches for questions such as adjusted models, propensity checks, subgroups, or sensitivity analyses.

It runs only after gates

Branch exploration requires a locked plan and readiness checks. It cannot start before the data and question are understood.

It has budget and stop controls

The queue records work items, progress, failure budget, stop events, and resume events so exploration does not expand without limits.

It does not rewrite primary conclusions

A branch must pass evaluation, promotion gates, evidence requirements, and user confirmation before it can become a formal plan amendment.

It leaves an audit trail

Branches, experiments, evaluations, promotion reviews, and plan amendments are written as project artifacts for report and audit review.

It is useful for follow-up analyses

Typical uses include adjusted models, ROC/AUC, propensity balance, learning-curve exploration, and sensitivity checks.

In one sentence: Auto Research does not let AI finish the study and announce the answer; it opens controlled side branches next to the locked plan, gathers supplemental evidence, and lets gates plus the user decide what can be promoted.

Key gate: concept alignment and plan registration require explicit confirmation. If Phase 8 later departs from the locked plan, the deviation must be logged.

MCP Tools

The tools are the workflow control surface

Major operations should go through RDE MCP tools so artifacts, readiness checks, and audit logs are produced consistently.

StageMain toolsPurpose

RDE MCP Server

Manages projects, pipeline state, artifact storage, decision/deviation logs, report readiness, and audit.

automl-stat-mcp

Optional heavy engine. Without Docker, RDE still uses local-lite statsmodels/scipy fallbacks; only unsupported heavy vendor workflows require automl.

Artifacts & Audit

Every judgment should be inspectable

RDE outputs a project-scoped evidence package, not only a written summary. Reports should point back to the phase, check, and log entry that support each claim.

Data-understanding artifacts

intake_report.json, schema.json, profile, and quality reports establish the data baseline.

Planning artifacts

concept_alignment.md, variable_roles.json, and analysis_plan.yaml make the question-to-method mapping reviewable.

Execution artifacts

Phase 8 analyses, figures, Table 1, model summaries, and fallback notes are saved under the project.

Audit artifacts

decision_log.jsonl, deviation_log.jsonl, report_readiness, and audit_report.json are the trust core.

Common Flows

How user requests map to governed paths

A quick overview, a group comparison, and a manuscript handoff use different levels of governance.

Full report

Phase 0-12: setup, intake, schema, alignment, plan, readiness, execution, collect, report, audit, improve, handoff.

Quick Explore

init_project -> run_intake -> build_schema -> profile_dataset -> quick report, explicitly marked as not fully audited.

Group comparison / Table 1

Complete Phase 0-7 first, then run generate_table_one or compare_groups with variable roles and assumptions checked.

Manuscript handoff

collect_results marks publishable items, assemble_report builds the report, run_audit scores it, and export_handoff packages evidence.

VSIX / Codex

Local-first startup; Docker is not the core requirement

VSIX and Codex support should start the bundled/local RDE MCP first. automl-stat-mcp is optional for heavier vendor workflows.

Codex MCP setup

python scripts/configure_codex_mcp.py --apply
python scripts/codex_rde_smoke.py --list-tools-only
python scripts/codex_rde_smoke.py

Manual RDE MCP startup

python3 -m pip install -e .
python3 -m rde
